# BingoEx (リアルタイム・ウェブビンゴゲーム)

BingoExは、参加者全員がブラウザから参加できるリアルタイムのオンラインビンゴゲームシステムです。
ホスト（主催者）が番号を抽選し、プレイヤー（参加者）の画面にリアルタイムで反映される仕組みを構築しています。

## 主な機能

### ホスト（主催者）向け機能
- **2 段階のセットアップ**: 景品数を決定した後、**景品名をテキストで入力する画面**に遷移します。
- **景品名の表示**: プレイヤーが当選するたびにホスト画面中央にモーダルで **景品名** と **当選者名** を大きく表示。
- **景品と当選者の一覧表示**: 各等の景品名と、誰が当選したかを常時一覧表示。
- **参加状況の管理**: 参加者名・接続状態・現在のビンゴ数をリアルタイムに確認可能。
- **抽選機能**: 重複しない1〜75までの番号をランダムに抽選。
- **リセット機能**: ゲーム状況の初期化（参加者のビンゴカードも再生成）。

### プレイヤー（参加者）向け機能
- **名前入力で簡単参加**: UUID を localStorage に保存するだけの気軽な参加方式。
- **ビンゴカードの自動生成**: 参加時に標準の 5x5 ビンゴカード（中央は FREE）を自動生成。
- **リアルタイム同期と判定**: 抽選結果が即座に反映され、ビンゴになれば「申告」ボタンから主催者に通知。
- **景品の通知**: 当選時に自分が獲得した景品名がポップアップで表示されます。

## 技術スタック

### フロントエンド (Client)
- **ライブラリ/フレームワーク**: React 19, Vite 6
- **ルーティング**: React Router v7
- **デザイン・UI**: モダンWebデザイン（CSS3, CSS Variables, アニメーション対応）
- **通信**: Socket.IO Client 4

### バックエンド (Server)
- **環境**: Node.js
- **フレームワーク**: Express 5
- **リアルタイム通信**: Socket.IO 4
- **その他**: `uuid`（プレイヤー識別用）, `cors`

## 環境構築と起動方法

本プロジェクトをローカルで動かすための手順です。
Node.js (LTS推奨) がインストールされていることを確認してください。

### 1. サーバーの起動

```bash
cd server
npm install
npm run dev
```

デフォルトでは `http://localhost:3001` で起動します。

### 2. クライアント（フロントエンド）の起動

別のターミナルで以下を実行してください。

```bash
cd client
npm install
npm run dev
```

ターミナルに表示されるローカルURL（例: `http://localhost:5173`）にブラウザでアクセスします。

## 遊び方

1. **ホスト画面を表示**: プロジェクター等に `http://localhost:5173/host` を表示します。
2. **景品数の設定**: 最初の画面で景品の数（例: 5）を入力します。
3. **景品名の入力**: 続けて、1 等から順に景品名をテキストで入力します。空欄の場合は「景品 N」が自動で設定されます。
4. **ゲーム開始**: 「ゲーム開始」ボタンで抽選画面へ遷移します。
5. **参加者の参加**: 参加者は各々の端末から `http://localhost:5173/` にアクセスし、名前を入力して参加します。
6. **抽選**: ホスト画面から「番号を引く」ボタンをクリックし、ビンゴ大会を進めます。
7. **当選の表示**: ビンゴになった参加者が「ビンゴを申告する」を押すと、ホスト画面中央に **獲得した景品名** と **当選者名** がモーダルで大きく表示されます。同時に景品リストの該当項目にも当選者が記録されます。
8. **終了**: 景品数の上限に達するとゲームは自動的に終了状態になります。

## 景品名を表示する仕組み (新機能)

- サーバー側（`server/index.js`）は `phase` を `setup → prizeInput → playing → finished` と遷移させます。
- `host:setPrizeCount` 受信で `phase = "prizeInput"` に移行し、ホスト画面で景品名入力フォームを表示します。
- `host:setPrizeNames` 受信で `phase = "playing"` になり、抽選が可能になります。
- プレイヤーが `player:claimBingo` を発行すると、サーバーは当選順の景品インデックスから景品名を割り当て、以下を送信します。
  - `host:newWinner` （ホスト画面へ。景品名と当選者名をモーダル表示）
  - `player:won` （当選者本人へ。景品名をモーダル表示）
- ホスト画面では景品一覧が常時表示され、当選済みの項目には当選者名がハイライト表示されます。
