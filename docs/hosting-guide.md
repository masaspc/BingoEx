## 1. 事前準備 (開催1週間前)

### 用意するもの

| 項目 | 例 | 備考 |
|------|-----|------|
| ConoHa VPS | 1GBプラン / Ubuntu 22.04 or 24.04 | 50名なら最小プランでも可 |
| ドメイン | `bingo.example.com` | サブドメイン推奨 |
| ホスト用パスワード | `MyBingo2026!` | ホスト画面へのログインに使用 |
| 景品リスト | 景品名と個数を事前に決定 | 最大50個まで設定可能 |

### DNS 設定

ドメインの **A レコード** を ConoHa VPS の IP アドレスに向けてください。

```
bingo.example.com.  A  xxx.xxx.xxx.xxx
```

> 反映に最大24時間かかる場合があるため、前日までに設定してください。

## 2. サーバーセットアップ (開催前日まで)

### ステップ 1: VPS に SSH 接続

```bash
ssh root@xxx.xxx.xxx.xxx
```

### ステップ 2: リポジトリ取得

```bash
git clone https://github.com/masaspc/BingoEx.git /opt/bingoex
cd /opt/bingoex
```

### ステップ 3: セットアップ実行 (自動)

```bash
export DOMAIN="bingo.example.com"
export HOST_PASSWORD="MyBingo2026!"
bash deploy/setup-conoha.sh
```

これだけで以下がすべて自動で完了します:

- Node.js 22 インストール
- PM2 (プロセス管理) インストール
- アプリのビルド
- Nginx (リバースプロキシ) 設定
- Let's Encrypt (SSL証明書) 取得
- ファイアウォール設定

### セットアップ完了後のURL

| 画面 | URL |
|------|-----|
| 参加者用 | `https://bingo.example.com/` |
| ホスト用 | `https://bingo.example.com/host` |

## 3. 動作確認 (開催前日)

### 3-1. サーバー状態の確認

```bash
ssh root@xxx.xxx.xxx.xxx
pm2 status
```

`bingoex` が `online` と表示されれば OK です。

### 3-2. ブラウザで動作テスト

1. **ホスト画面** (`https://bingo.example.com/host`) を PC で開く
2. ホスト用パスワードを入力してログイン
3. 景品数を「2」と入力して「次へ」
4. 適当な景品名を入力して「ゲーム開始」
5. 「番号を引く」を何度か押して番号が表示されることを確認
6. **参加者画面** (`https://bingo.example.com/`) をスマホで開く
7. 名前を入力して「参加する」
8. ビンゴカードが表示され、引いた番号がマークされることを確認
9. 確認が終わったら「リセット」を押してゲームを初期化

## 4. 当日の進行手順

### 開始前 (参加者集合前)

1. **ホスト画面** を PC (FullHD推奨) で開いてログイン
2. プロジェクター等に映す場合は、ホスト画面をそのまま表示

### 参加者の案内

参加者に以下を伝えてください:

> **スマホで下記URLにアクセスして、お名前を入力してください。**
>
> `https://bingo.example.com/`

- QRコードを事前に作成して配布すると便利です
- Wi-Fi環境があれば案内してください

### ゲーム設定 (ホスト画面で操作)

#### ステップ 1: 景品数の設定

1. 景品の個数を入力
2. 「次へ（景品名の入力）」をクリック

#### ステップ 2: 景品名の入力

1. 1等から順に景品名を入力
   - 空欄の場合は「景品 N」と自動表示されます
2. 「ゲーム開始」をクリック

> この時点で参加者の画面にもゲーム開始が反映されます。

### 抽選の進行

1. **「番号を引く」ボタンをクリック** → 番号がランダムに抽選されます
2. ホスト画面に大きく番号が表示されるので、参加者に読み上げます
3. 参加者のスマホには自動的にマークが付きます
4. **ビンゴになった参加者**は画面の「ビンゴを申告する」ボタンを押します
5. ホスト画面に **当選者のポップアップ** が表示されます（景品名と当選者名）
6. ポップアップを確認したら「閉じる」を押して次の抽選へ

### 当選の流れ

```
参加者がビンゴ達成
    ↓
参加者が「ビンゴを申告する」ボタンを押す
    ↓
サーバーがビンゴを検証 (不正チェック)
    ↓ 検証OK
ホスト画面にポップアップ表示:
  「N等 当選！ ○○さん — 獲得景品: △△」
    ↓
参加者画面にも当選モーダル表示:
  「おめでとうございます！ あなたが獲得した景品は △△」
```

> サーバー側でビンゴの成立を検証するため、不正な申告はできません。

### すべての景品が確定したら

- 自動的にゲームが「終了」フェーズに移行します
- ホスト画面に **結果サマリー** が表示されます

## 5. 終了後の作業

### 5-1. 当選者リストの出力

ゲーム終了後、ホスト画面に表示される結果サマリーから:

- **「CSV ダウンロード」** → Excel等で開けるCSVファイルをダウンロード
- **「テキストをコピー」** → クリップボードにテキスト形式でコピー

#### CSV の内容

```csv
順位,景品名,当選者名,当選時刻
1,"Nintendo Switch","山田 太郎",2026-04-21T15:30:45.123Z
2,"お米10kg","佐藤 花子",2026-04-21T15:35:12.456Z
3,"Amazonギフト券","田中 一郎",2026-04-21T15:40:33.789Z
```

> ゲーム進行中でも「結果一覧」ボタンからCSVダウンロード可能です。

### 5-2. 景品の配布

CSVまたはホスト画面の当選者リストを元に、景品を配布してください。

### 5-3. ゲームのリセット (必要な場合)

もう一度ゲームを行う場合は「リセット」ボタンを押してください。
参加者はそのままで、カードと抽選結果が初期化されます。

### 5-4. サーバーの後処理

イベント終了後、VPSを維持する必要がなければ:

```bash
ssh root@xxx.xxx.xxx.xxx

# サーバーを停止する場合
pm2 stop bingoex

# VPS自体を削除する場合は ConoHa コントロールパネルから
```

## 6. トラブルシューティング

### 参加者が接続できない

- WiFi環境を確認してください
- `https://bingo.example.com/` の `https` が必要です（`http` ではなく）
- DNS が反映されていない可能性があります → `ping bingo.example.com` で確認

### ホスト画面にログインできない

- パスワードを確認してください
- セットアップ時に設定した `HOST_PASSWORD` の値です
- パスワードを変更する場合:

```bash
ssh root@xxx.xxx.xxx.xxx
cd /opt/bingoex
HOST_PASSWORD="新しいパスワード" pm2 restart bingoex --update-env
pm2 save
```

### サーバーが応答しない

```bash
ssh root@xxx.xxx.xxx.xxx

# プロセス状態を確認
pm2 status

# ログを確認
pm2 logs bingoex --lines 50

# 再起動
pm2 restart bingoex
```

### 画面が表示されるがリアルタイム更新されない

WebSocket接続の問題です。以下を確認してください:

```bash
# Nginx の設定を確認
nginx -t

# Nginx を再起動
systemctl restart nginx
```

### 「同一ネットワークからの参加上限に達しています」と表示される

同じIPから3名以上が参加しようとした場合に表示されます。
会場のWiFiなど全員が同一IPになる環境では、上限を引き上げてください:

```bash
MAX_CONNECTIONS_PER_IP=60 pm2 restart bingoex --update-env
pm2 save
```

### アプリを更新したい

```bash
ssh root@xxx.xxx.xxx.xxx
bash /opt/bingoex/deploy/update.sh
```

## 7. 環境変数一覧

| 変数名 | デフォルト | 説明 |
|--------|-----------|------|
| `PORT` | `3001` | サーバーのポート番号 |
| `HOST_PASSWORD` | (空=無効) | ホスト画面のパスワード。**本番では必ず設定** |
| `MAX_PLAYERS` | `100` | 最大参加者数 |
| `MAX_CONNECTIONS_PER_IP` | `3` | 同一IPからの新規参加上限 |
| `NODE_ENV` | `development` | `production` で本番モード |
| `CLIENT_ORIGIN` | `*` | 開発時のCORS許可オリジン (本番では不使用) |

### 環境変数の変更方法 (PM2)

```bash
# 例: 同一IP制限を緩和して再起動
MAX_CONNECTIONS_PER_IP=60 HOST_PASSWORD="MyBingo2026!" pm2 restart bingoex --update-env
pm2 save
```
